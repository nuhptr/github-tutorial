### Different using rebase and merge

```sh
- Rebase akan terlihat sangat rapih, karena timelinenya seakan
terlihat hanya satu timeline pada repository

- namun, rebase sebenarnya secara otomatis menulis ulang semua commit yang kita lakukan,
dalam artian commit id pasti berubah, artinya semua referensi ke commit id
sebelum-sebelumnya akan rusak dan hilang

- tidak ada mana yang lebih baik, semua tergantung kebutuhan
```
